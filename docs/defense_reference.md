# Rao2016 图像篡改检测 Demo 答辩参考

本文档用于答辩和报告撰写，说明原论文的主要思想、本 demo 借鉴了什么、简化了什么，以及老师可能追问的问题。

## 1. 选题背景

本实验属于“信息内容安全”中的内容真实性与数字取证方向。现实中图像可能被拼接、复制移动、局部修补或后期处理，肉眼有时很难判断。图像篡改检测的目标不是识别图中具体物体，而是判断图像是否存在不自然的取证痕迹，例如边界残差、压缩不一致、局部纹理统计异常等。

Rao2016 论文关注两类常见篡改：

- `splicing`：把一张图像中的区域拼接到另一张图像中。
- `copy-move`：把同一张图像中的一块区域复制到另一位置，用来遮挡或伪造内容。

这两类篡改都可能破坏图像局部统计一致性，因此可以用 patch 级别的 CNN 学习局部篡改痕迹。

## 2. 原论文主要思想

论文题目是 `A Deep Learning Approach to Detection of Splicing and Copy-Move Forgeries in Images`。它的核心思想可以概括为：

```text
图像 -> 局部 patch -> CNN 特征提取 -> patch 特征融合 -> SVM 判断 authentic/forged
```

### 2.1 Patch-based 检测

原论文不是直接把整张图像输入网络，而是从图像中裁剪 `128 x 128 x 3` 的局部 patch。这样做有两个原因：

- 篡改痕迹通常是局部的，例如拼接边界、复制区域边缘、局部噪声不一致。
- 不同图片尺寸不统一，patch 输入更容易训练固定结构的 CNN。

训练时，论文从篡改边界附近采样正样本 patch，从真实图像中采样负样本 patch。测试时，用滑动窗口扫描整张图像，得到多个 patch 的特征。

### 2.2 CNN 自动学习取证特征

传统图像取证方法常依赖人工设计特征，例如噪声统计、压缩痕迹、边缘不连续性。Rao2016 使用 CNN 自动学习层次化特征，让模型从数据中学习真实图像与篡改图像的差异。

原论文网络输入为 `128 x 128 x 3` patch，整体结构包含多个卷积层、池化层和全连接层，最后输出二分类结果。

### 2.3 SRM 高通滤波器初始化

论文最值得讲的创新点是：第一层卷积核不是完全随机初始化，而是使用 SRM 的 30 个高通滤波器初始化。

SRM 高通滤波器的作用是突出图像残差。普通 RGB 像素里包含大量内容信息，例如物体、颜色、纹理和光照，这些内容信息可能干扰篡改检测。高通残差会压制图像语义内容，放大不自然的细微痕迹，例如边缘断裂、噪声不连续、局部统计异常。

答辩时可以这样说：

> 原论文希望 CNN 关注的不是“图中是什么”，而是“图像生成过程是否一致”。SRM 初始化相当于给网络一个取证先验，让第一层更偏向残差信号。

### 2.4 特征融合与 SVM 分类

测试阶段，论文会用滑动窗口在一张测试图上提取多个 patch 特征。因为最终任务是整图 authentic/forged 分类，所以需要把多个 patch 特征融合成一张图的特征。论文使用 mean pooling 和 max pooling 等方式聚合 patch 特征，然后用 SVM 完成最终二分类。

## 3. 本 Demo 借鉴了什么

本 demo 的目标不是完整复现论文全部工程细节，而是复现最核心的实验逻辑：用 CNN 对真实图像和篡改图像做二分类。

### 3.1 借鉴 patch-based CNN 思路

本 demo 同样使用 `128 x 128` 输入尺寸。训练时通过 resize、crop、normalize 得到固定大小图像输入，保持了原论文 patch 输入的基本设定。

对应代码：

- `src/forgery_exp/data.py`：定义图像变换、数据加载、类别目录。
- `src/forgery_exp/model.py`：定义轻量 CNN。

### 3.2 借鉴二分类任务设置

本 demo 保留 authentic / forged 二分类：

```text
authentic = 0
forged = 1
```

数据目录也按二分类任务组织：

```text
dataset/
  train/
    authentic/
    forged/
  val/
    authentic/
    forged/
  test/
    authentic/
    forged/
```

### 3.3 借鉴局部篡改痕迹学习

合成数据中的 forged 图像会复制或拼接一个矩形区域，用来模拟 copy-move / splicing 的局部边界。模型训练时并不知道篡改区域在哪里，只能根据整体图像中的局部统计差异学习分类。

对应代码：

- `src/forgery_exp/make_toy_dataset.py`：生成 authentic 和 forged 合成图像。

### 3.4 借鉴实验评价方式

本 demo 输出训练 loss、验证 accuracy、测试 accuracy、混淆矩阵和预测样例。这些指标适合课程报告展示模型是否学到了二分类能力。

对应输出：

```text
runs/rao2016_simple/
  best_model.pt
  metrics.json
  loss_accuracy.png
  confusion_matrix.png
  sample_predictions.png
```

## 4. 本 Demo 做了哪些简化

### 4.1 没有完整使用 CASIA / Columbia 数据集

原论文使用 CASIA v1.0、CASIA v2.0、Columbia DVMM 等公开篡改数据集。本 demo 为了保证环境可运行，先提供了合成玩具数据生成脚本。合成数据能验证流程，但不能代表真实篡改检测性能。

答辩表述：

> 由于真实数据集下载和整理成本较高，本实验先用合成数据完成代码复现和流程验证。若进一步完善，可以替换为 CASIA v2.0，并保持同样的目录结构训练。

### 4.2 没有按篡改边界采样正样本 patch

原论文依赖篡改 mask 或边界信息，从篡改边界附近采样 patch。本 demo 只做图像级 authentic/forged 标签，没有使用篡改区域 mask，因此没有精确边界采样。

影响：

- 优点：数据准备简单，只需要二分类目录。
- 缺点：模型学习信号更粗，不能保证每个训练样本都包含明显篡改边界。

### 4.3 没有 SRM 高通滤波器初始化

原论文第一层使用 30 个 SRM 高通滤波器初始化。本 demo 的卷积层使用 PyTorch 默认初始化，没有手工加入 SRM 滤波器。

原因：

- SRM 滤波器实现和初始化细节会增加复现复杂度。
- 课程 demo 重点展示 CNN 二分类流程，而不是完整还原取证特征工程。

答辩时不要说“完全复现了 SRM”，应该说：

> 本 demo 借鉴了原论文关注局部残差和篡改痕迹的思想，但没有实现 SRM 初始化。这是后续可以改进的部分。

### 4.4 没有滑动窗口扫描整图

原论文测试时对整张图像做滑动窗口，提取多个 patch 特征。本 demo 直接把图像 resize/crop 到固定尺寸后送入 CNN，相当于做一个简化的图像级分类。

影响：

- 优点：实现简单，训练和推理速度快。
- 缺点：如果篡改区域很小，直接 resize/crop 可能丢失关键局部线索。

### 4.5 没有使用 CNN 特征 + SVM

原论文用 CNN 提取特征，再用 pooling 聚合，最后用 SVM 分类。本 demo 直接在 CNN 末尾接 `Linear` 层输出二分类。

这相当于把特征提取和分类器合并成端到端训练流程。它更符合常见 PyTorch 教学实现，但和原论文的最终分类器不同。

### 4.6 网络结构更轻量

原论文使用更深的 CNN。本 demo 使用轻量网络：

```text
Conv2d(3, 30, 5) + BN + ReLU
Conv2d(30, 30, 5) + BN + ReLU + MaxPool
Conv2d(30, 16, 3) + BN + ReLU
Conv2d(16, 16, 3) + BN + ReLU + MaxPool
Conv2d(16, 32, 3) + BN + ReLU
AdaptiveAvgPool2d
Linear -> 2 classes
```

这样设计的理由是：

- 能在 CPU 环境快速训练。
- 保留卷积、池化、非线性和分类头这些核心组件。
- 适合课程演示，不追求论文级性能。

## 5. Demo 流程怎么讲

答辩可以按下面顺序讲：

1. 研究问题：判断图像是真实还是篡改。
2. 原论文方法：patch CNN 学习局部篡改痕迹，SRM 初始化突出残差，滑窗提取 patch 特征，pooling 后用 SVM 分类。
3. 本实验实现：使用 PyTorch 搭建轻量 CNN，数据分为 authentic/forged，训练二分类器。
4. 简化原因：课程复现以流程可运行为目标，省略真实数据集、SRM 初始化、滑动窗口和 SVM。
5. 实验输出：loss 曲线、accuracy 曲线、混淆矩阵、预测样例。
6. 局限与改进：加入 CASIA v2.0、SRM 初始化、mask 边界采样、滑动窗口、多尺度检测。

## 6. 可能被问到的问题

### Q1：为什么图像篡改检测不能只看语义内容？

因为篡改检测关注的是图像形成过程是否一致，而不是图中物体是什么。两张图语义上都可能很合理，但其中一张可能存在拼接边界、噪声不一致或压缩痕迹异常。因此取证模型更应该关注低层纹理、残差和局部统计特征。

### Q2：SRM 高通滤波器有什么作用？

SRM 高通滤波器用于提取图像残差。它会削弱大范围颜色和物体内容，突出细微变化。对于篡改检测来说，这些残差信号往往比图像语义更有用。

### Q3：本 demo 是否完整复现了 Rao2016？

没有。本 demo 是简化复现，复现的是“patch-based CNN 做 authentic/forged 二分类”的核心思想。没有完整复现 SRM 初始化、滑动窗口整图扫描、patch pooling 和 SVM 分类。

### Q4：为什么不用 SVM，而是直接用 Linear 分类？

为了简化工程流程。Linear 分类头可以和 CNN 端到端训练，代码更短，也更适合 PyTorch 教学实验。SVM 是原论文流程的一部分，后续可以把 CNN 倒数层特征导出，再训练 SVM 做对比实验。

### Q5：为什么用合成数据？

真实篡改数据集下载、清洗、划分比较耗时。合成数据可以快速验证数据加载、模型训练、指标输出和预测流程。它不能替代真实数据集上的性能评估，但适合作为课程 demo 的最小可运行版本。

### Q6：如果老师问准确率为什么不高，怎么回答？

可以回答：

> 当前结果主要用于验证流程可运行。由于使用的是小规模合成数据，并且训练 epoch 较少，准确率不能代表模型上限。正式实验应增加训练轮数、扩大数据规模，最好换成 CASIA v2.0 等真实数据集。

### Q7：本实验相比原论文最大的差距是什么？

最大差距有三个：

- 没有使用真实公开数据集和篡改 mask。
- 没有实现 SRM 高通滤波器初始化。
- 没有做滑动窗口 patch 特征融合和 SVM 分类。

### Q8：如果继续改进，优先做什么？

优先级建议：

1. 使用 CASIA v2.0 替换合成数据。
2. 加入真实篡改 mask，按边界采样 patch。
3. 在第一层加入 SRM 高通滤波器初始化。
4. 实现滑动窗口扫描整图，并对 patch 特征做 mean/max pooling。
5. 导出 CNN 特征，训练 SVM，与端到端 Linear 分类头比较。

## 7. 一分钟答辩稿

本实验复现的是 Rao2016 图像篡改检测论文的简化版本。原论文主要用于检测 splicing 和 copy-move 两类图像篡改，它的核心思想是把图像裁成 `128 x 128` 的 patch，用 CNN 学习局部篡改痕迹。论文中特别重要的一点是第一层卷积使用 SRM 高通滤波器初始化，这样可以抑制图像内容本身，突出残差和边界异常。测试时，原论文用滑动窗口提取多个 patch 特征，再通过 pooling 聚合成整图特征，最后用 SVM 判断真实或篡改。

我的 demo 保留了最核心的 CNN 二分类流程：把数据整理成 authentic 和 forged 两类，输入固定大小图像，用轻量 CNN 训练二分类模型，并输出 loss 曲线、accuracy 曲线、混淆矩阵和预测样例。为了降低课程实验复杂度，我没有完整复现 SRM 初始化、滑动窗口扫描和 SVM 分类，也没有强制使用 CASIA 数据集，而是提供了合成数据生成脚本来验证完整流程。后续如果继续完善，可以换成 CASIA v2.0，加入 mask 边界采样、SRM 初始化和 CNN 特征加 SVM 的原论文流程。
